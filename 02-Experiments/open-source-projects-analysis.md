## 二、OpenClaw：核心研究平台 ⭐

### 2.1 项目档案

| 维度 | 信息 |
|------|------|
| **GitHub** | https://github.com/openclaw/openclaw |
| **Stars** | 355K+（2026年4月，史上增长最快的开源AI Agent项目之一） |
| **定位** | 自托管个人AI助手，运行于自有设备 |
| **运行实例** | **Jarvis目前就运行在OpenClaw之上** |
| **语言** | TypeScript / Node.js |
| **发布历史** | 2025年11月：Clawdbot → 2026年1月：两次更名 → OpenClaw |

### 2.2 核心架构

OpenClaw的架构天然就是一个多智能体协作系统：

```
OpenClaw Gateway（中央协调器）
  ├── 主Agent（Jarvis本体）
  │     ├── Skills系统（技能模块化）
  │     ├── 记忆系统（memory/*.md / MEMORY.md）
  │     └── 消息通道（50+平台）
  │
  ├── Subagents（子Agent）
  │     ├── spawn（隔离会话运行）
  │     ├── sessions_spawn（isolated模式）
  │     └── cron（定时触发）
  │
  └── 通信层
        ├── Gateway Token（安全认证）
        ├── MCP Server（工具扩展）
        └── Skills（可复用能力模块）
```

### 2.3 多智能体能力详解

**① Subagents（子Agent）**
- 通过`sessions_spawn`启动隔离子Agent会话
- 支持`runtime=subagent` / `runtime=acp`两种模式
- 子Agent继承父级工作目录，自动文件系统访问
- 典型用途：并行处理多路信息检索、并行写作任务

**② Agent Teams（多Agent团队编排）**
- 支持大规模多Agent工作流
- 可配置角色（role）和任务路由
- 通过`skill`系统实现角色定义与能力注入
- 规模由上下文限制决定（非固定上限）

**③ Skills（技能系统）**
- OpenClaw的Skill = 可插拔的多Agent能力模块
- 支持条件触发（context-aware）
- SKILL.md定义技能规范，运行时自动加载
- Jarvis当前安装**50+ Skills**，涵盖各个研究领域

### 2.4 在研究体系中的位置

```
033研究体系
└── OpenClaw ← 这是Jarvis的身体，也是研究对象
    ├── ① 运行平台：所有实验最终部署在OpenClaw上
    ├── ② 基准实现：Jarvis的多智能体架构是最直接的参考
    ├── ③ Skills系统：多Agent角色定义与协作的实践范本
    └── ④ 自进化机制：Jarvis的HEARTBEAT + 判断库 + 教训库 = OpenClaw上的自进化实现
```

**核心价值：** OpenClaw是目前唯一在**生产环境运行 Jarvis**的多智能体平台，其架构本身既是研究成果，也是研究工具。

## 三、Hermes Agent（Nous Research）：自进化个体Agent ⭐⭐

### 3.1 项目档案

| 维度 | 信息 |
|------|------|
| **GitHub** | https://github.com/nousresearch/hermes-agent |
| **开发方** | Nous Research |
| **定位** | 自我改进型AI Agent——"与你一同成长的Agent" |
| **核心特性** | 持久记忆 · 自主技能创建 · 自我改进 · 多平台 |

### 3.2 核心架构

**三大自进化机制：**

```
Hermes Agent架构
├── 持久记忆系统（Persistent Memory）
│     ├── Agent自主管理记忆内容（agent-curated memory）
│     ├── 周期性"轻推"机制（periodic nudges）——防止记忆漂移
│     └── 跨会话记忆保留
│
├── 自主技能创建（Autonomous Skill Creation）
│     ├── 从实践经验中自动生成Skills
│     ├── 每次任务执行后自动改进已有Skill
│     └── 不需要人工干预技能开发
│
└── 自进化训练（Self-Improvement）
      ├── DSPy框架（程序优化）
      ├── GEPA（通用参数优化）
      ├── 基于真实执行轨迹自动优化Skill文件
      └── 无需GPU，仅需API调用
```

### 3.3 与OpenClaw的关键差异

| 维度 | Hermes Agent | OpenClaw |
|------|-------------|---------|
| **记忆方向** | 个体Agent自主管理 | 系统级记忆文件 |
| **技能创建** | 自动从经验生成 | 手动/SKILL.md定义 |
| **自进化** | 基于DSPy+GEPA自动优化 | 通过Cron+教训库手动迭代 |
| **多Agent** | 单体为主（路线图有多Agent） | 原生多Agent编排 |
| **调度方式** | 自然语言Cron | 显式Cron配置 |

### 3.4 在研究体系中的位置

```
033研究体系
├── Agent个体研究
│     └── Hermes Agent ← 自我进化机制的最佳参考
│           ├── 自主技能创建 → Jarvis Skills自动化方向参考
│           ├── 记忆管理机制 → 多Agent共享记忆参考
│           └── 自进化路径 → Jarvis自进化的终极形态
```

**关键启示：** Jarvis的自进化当前依赖"手动经验积累+教训库"；Hermes展示了"基于DSPy的自动化自进化"路径——这是值得跟进的研究方向。

## 四、OWL（CAMEL-AI）：多智能体协作学术框架 ⭐⭐

### 4.1 项目档案

| 维度 | 信息 |
|------|------|
| **GitHub** | https://github.com/camel-ai/owl |
| **论文** | arXiv:2505.23885 · NeurIPS 2025 |
| **框架基础** | 构建于CAMEL-AI Framework之上 |
| **定位** | 层次化多智能体任务自动化框架 |

### 4.2 核心创新：WORKFORCE框架

OWL提出了**WORKFORCE（优化劳动力学习）**框架，核心设计思想：

**层次化解耦：**
```
战略规划层（Strategic Planner）
  └── 领域执行层（Domain-Specific Executors）
        ├── Executor-A（执行器A）
        ├── Executor-B（执行器B）
        └── ...（可扩展）
```

- **战略规划层**：Agent将复杂任务解耦为子任务序列，专注"做什么"
- **领域执行层**：各Executor专注于特定领域的精确执行，专注"怎么做"
- 两者通过结构化通信协议交互，降低全局协调复杂度

**关键创新点：**
1. **领域无关规划器优化**：使用强化学习优化Planner，使其能够泛化到新领域
2. **跨领域泛化**：Planner不针对特定领域训练，但能泛化到任意领域任务
3. **稀疏通信拓扑**：Planner与Executor之间只需传递任务/结果，不需要全连接通信

### 4.3 与Anthropic架构的对比

| 维度 | OWL（WORKFORCE） | Anthropic多Agent研究系统 |
|------|-----------------|----------------------|
| **拓扑** | 层次化（Planner→Executor） | 主从（Master→Sub-agent） |
| **通信** | 稀疏（仅任务/结果传递） | 中等（子Agent间可通信） |
| **学习** | RL优化Planner | 固定角色，无在线学习 |
| **泛化** | 强（跨领域） | 弱（针对研究任务定制） |
| **学术深度** | 高（NeurIPS论文） | 中（工程博客，无正式论文） |
| **生产成熟度** | 低（学术框架） | 高（Anthropic内部生产） |

### 4.4 在研究体系中的位置

```
033研究体系
├── 协作架构研究
│     └── OWL ← 学术验证的层次化协作模式
│           ├── 战略规划与执行分离 → 实验1通信效率的"层次拓扑"参考
│           ├── WORKFORCE论文 → 实验设计的理论基础
│           └── 稀疏通信拓扑 → 降低通信开销的理论支撑
```

**关键启示：** OWL的WORKFORCE框架为实验1（通信效率对比）中的"层次拓扑"提供了学术背书——战略规划与执行分离可以显著降低通信量。

## 五、MiroFish：群体智能预测引擎 ⭐

### 5.1 项目档案

| 维度 | 信息 |
|------|------|
| **GitHub** | https://github.com/666ghj/MiroFish |
| **Stars** | 较小（早期开源项目） |
| **定位** | 多智能体群体智能预测仿真引擎 |
| **核心思想** | 模拟真实世界生成数千个AI Agent，预测复杂系统演化 |

### 5.2 核心工作原理

```
真实世界数据输入
  ├── 新闻、报告、文献、社交媒体等
  └── 任何包含社会动态信息的数据源
        ↓
MiroFish 引擎
  ├── 从真实数据中提取种子信息（seed information）
  ├── 生成数千个AI Agent（模拟社会个体）
  ├── 每个Agent拥有独立角色、目标、行为规则
  └── Agent间交互 → 社会动力学模拟
        ↓
预测输出
  ├── 复杂系统未来演化趋势
  ├── 替代方案效果评估
  └── 政策/决策影响仿真
```

**独特价值：**
- 不是让Agent"完成特定任务"，而是让Agent"模拟真实世界"
- 适用于：金融风险预测、政策评估、舆情演化、社会影响分析
- 展示了多智能体系统的另一个维度：**预测性仿真**

### 5.3 在研究体系中的位置

```
033研究体系
├── 涌现行为研究
│     └── MiroFish ← 群体智能与预测仿真的实践案例
│           ├── 数千Agent规模实验 → 问题G（涌现行为）的大规模验证
│           ├── 社会动力学模拟 → 问题F（博弈论协商）的宏观版本
│           └── 预测性仿真 → 未来Jarvis×多智能体的应用方向
```

**关键启示：** MiroFish代表了一种新的多智能体应用范式——**预测性仿真**。如果将Wayne的专业领域（网络系统、互联网架构）结合MiroFish的思路，可以构建"网络行为预测"的多智能体模拟引擎。

## 七、研究优先级建议

基于四个项目的成熟度和研究价值，建议如下研究顺序：

### 第一优先：深入OpenClaw多智能体架构（立即可做）
**行动：** 以Jarvis为研究对象，解构OpenClaw的多智能体编排机制
**产出：** 内部研究报告：《OpenClaw多Agent架构深度解析》
**价值：** 这是唯一在生产环境运行的项目，结果可直接用于实验设计

### 第二优先：Hermes Agent记忆系统对比（2—3周）
**行动：** 在OpenClaw上实现类似Hermes的"Agent自主记忆管理"机制
**产出：** 记忆系统改进方案
**价值：** Jarvis的自进化效率可以通过引入类似DSPy的自动优化机制提升

### 第三优先：OWL的WORKFORCE框架复现（1—2个月）
**行动：** 在OpenClaw上实现层次化多Agent（Planner+Executor）
**产出：** 实验1的"层次拓扑"实现
**价值：** 为实验1提供学术支撑，同时产出可发表的学术成果

### 第四优先：MiroFish的预测性仿真探索（3个月+）
**行动：** 探索将MiroFish思路引入Wayne专业领域（网络系统预测）
**产出：** "网络行为预测多智能体引擎"提案
**价值：** 高度创新的应用方向，与Wayne研究领域直接结合
