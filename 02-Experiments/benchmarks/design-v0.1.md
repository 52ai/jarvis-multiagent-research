## 🎯 实验目标

**一句话**：在中文长尾检索 + 总结任务上，量化"Agent 数量 vs 任务质量 vs 成本"的关系曲线。

**核心假设**（H1-H3）：
- **H1**：3-5 Agent 是中文任务的最优协作规模（验证 Google 结论）
- **H2**：协作开销 ≥ 3 Agent 时显著上升
- **H3**：主从架构（1 + N）比平等架构（N×N）效率高

---

## 📋 任务定义

### 任务类型：Web 检索 + 综合报告

**为什么选这个**：
1. 真实场景（每个 AI Agent 工程师都会用）
2. 易评估（输出是结构化报告，可用 LLM-as-judge）
3. 成本可控（单次运行 Token 1K-5K）

### 5 个固定测试集（避免单点偏差）

| ID | 主题 | 难度 | 预期 Agent 数 |
|----|------|------|--------------|
| T1 | 多智能体协作技术综述 | ⭐⭐ | 3 |
| T2 | 2026 年 LLM 推理优化综述 | ⭐⭐⭐ | 4 |
| T3 | A2A 协议 vs MCP 协议对比 | ⭐⭐ | 3 |
| T4 | AI Agent 安全威胁全景 | ⭐⭐⭐⭐ | 5 |
| T5 | 多智能体涌现行为研究 | ⭐⭐⭐⭐⭐ | 5 |

每个任务跑 3 次取中位数（控制随机性）。

## 🏗 实验矩阵

### 对照组 1：单 Agent 基线

- 1 个 Agent 完成全部工作
- 无协作开销

### 实验组 1：平等架构（n×n 全连接）

- n ∈ {2, 3, 5, 7, 9}
- 每个 Agent 都能看到所有人的消息
- 模拟"群聊"模式

### 实验组 2：主从架构（Anthropic 风格）

- 1 主 Agent（Opus 4）+ n-1 子 Agent（Sonnet 4）
- n ∈ {2, 3, 5, 7}
- 主 Agent 编排 + 子 Agent 并行

### 实验组 3：流水线架构（顺序传递）

- n ∈ {3, 5}
- 检索 → 过滤 → 综合 → 校对 → 输出
- 模拟工厂流水线

**总实验数**：1（基线）+ 5（平等）+ 4（主从）+ 2（流水线）= **12 组**
**每组 5 任务 × 3 次** = **180 次实验**
**预计总 Token**：~500K（单 Agent 基线）+ ~5M（实验组）= **~5.5M tokens**

## 📊 评估指标

### 1. 效率指标

| 指标 | 定义 | 单位 |
|------|------|------|
| **Latency** | 端到端完成时间 | 秒 |
| **Token Cost** | 总 Token 数（含输入输出） | 千 Token |
| **Communication Overhead** | Agent 间消息 Token / 总 Token | % |
| **Cost per Quality Point** | Token Cost / 质量分 | Token / pt |

### 2. 质量指标

| 指标 | 评估方法 | 满分 |
|------|---------|------|
| **事实准确性** | LLM-as-judge + 关键事实清单 | 10 |
| **结构完整度** | 模板匹配度（标题/章节/数据） | 10 |
| **可读性** | LLM-as-judge | 10 |
| **洞察深度** | LLM-as-judge | 10 |
| **综合分** | 加权平均（40/20/20/20） | 10 |

### 3. 鲁棒性指标

| 指标 | 评估方法 |
|------|---------|
| **任务成功率** | 5 次中成功 ≥ 3 次的比例 |
| **输出方差** | 3 次运行质量的 std |
| **超时率** | 超 60s 未完成的次数 / 总次数 |

## 🤖 Agent 实现

### 平台：OpenClaw 自身 + OWL 框架

- **OpenClaw**：主 Agent 运行环境（自身）
- **OWL**（CAMEL-AI）：参考的多 Agent 协作框架
- **模型**：Opus 4（主）+ Sonnet 4（子）
- **温度**：0.7（控制随机性）
- **最大轮数**：20

### Agent 角色

```
检索 Agent        - 负责 web 检索
过滤 Agent        - 负责去重 + 质量过滤
综合 Agent        - 负责跨源整合
校对 Agent        - 负责事实核查
主编 Agent        - 负责结构化输出
```

## 📈 预期结论（先验）

如果 v1.0 报告的 5 条核心发现成立，预期结果：

| 指标 | 1 Agent | 3 Agent | 5 Agent | 7 Agent |
|------|---------|---------|---------|---------|
| Token Cost | 1.0× | ~1.5-2× | ~2-3× | ~3-4× |
| 质量分 | 6.0 | **7.5** | **8.0** | 7.5（开始降） |
| 通信开销 | 0% | 15-25% | 30-45% | 50-65% |
| 成本/质量点 | 0.17 | 0.27 | 0.38 | 0.50 |

**核心发现预测**：
- 3-5 Agent 是甜区
- 7+ Agent 通信开销 > 50%
- 主从架构比平等架构节省 30-40% Token

## ⚠️ 风险控制

| 风险 | 缓解策略 |
|------|---------|
| **Token 成本失控** | 单次实验 Token 上限 50K；超出则降级任务 |
| **OWL 集成失败** | Phase 2 预留 2 周降级方案（用 OpenClaw 内置 multi-agent 模式） |
| **LLM-as-judge 偏差** | 用 Opus 4 作为 judge，与 Sonnet 4 评分对比 |
| **任务集偏差** | 5 个任务覆盖多难度；后续 Phase 3 扩展到 10 个 |
| **API 限流** | 实验间隔 5s，重试机制 3 次 |
