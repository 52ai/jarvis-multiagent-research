## 2 详细时间线

### 2.1 Era 1: 网络互联（1969-1995）—— 28 年

| 年份 | 事件 | 影响 |
|------|------|------|
| 1969 | ARPANET 4 节点 | 网络互联起点 |
| 1974 | TCP/IP 协议 (Cerf & Kahn) | 互联基础 |
| **1983** | **DNS 诞生**（Mockapetris RFC 882）| **解决 hosts 文件 N² 问题** |
| 1988 | DNS 扩展（RFC 1034/1035）| 全球可扩展 |
| 1995 | 商业互联网元年 | DNS 全球根 13 服务器 |

**关键洞察**：DNS 用 5 年时间（1983-1988）从 1 个 RFC 演化为全球基础设施。**这是 Agent Internet 应该效仿的演化速度**。

### 2.2 Era 2: Web 协议（1995-2010）—— 15 年

| 年份 | 事件 | 影响 |
|------|------|------|
| 1996 | HTTP/1.0 (RFC 1945) | 文本协议 |
| 1999 | SOAP 1.1 | 早期服务化 |
| 2000 | REST 概念 (Roy Fielding) | 架构风格 |
| 2006 | SOAP 1.2 (W3C) | 企业级服务化 |

**关键洞察**：Web 协议经历 15 年才稳定。**智能体协议若想稳定，需 2024-2030 整个 6 年窗口期**。

### 2.3 Era 3: API 经济（2010-2020）—— 10 年

| 年份 | 事件 | 影响 |
|------|------|------|
| 2010 | AWS API Gateway | API 商品化 |
| 2012 | RESTful API 设计规范 | 行业共识 |
| 2015 | GraphQL (Facebook) | 查询优化 |
| 2018 | gRPC (Google) | 高性能 RPC |
| 2019 | OpenAPI 3.0 | 标准化 |

**关键洞察**：API 经济 10 年成熟。**智能体 API 应该建立在 OpenAPI/JSON-RPC 之上**，而非全新协议。

### 2.4 Era 4: LLM 基础（2020-2023）—— 3 年

| 年份 | 事件 | 影响 |
|------|------|------|
| 2020 | GPT-3 发布 | 大模型时代 |
| 2021 | Codex / Copilot | 代码智能体 |
| 2022 | ChatGPT | 大众化 |
| **2023** | **Function Calling**（OpenAI）| **智能体能力起点** |

**关键洞察**：Function Calling 用了 3 年成为主流。**这是 MCP 的前身**——Anthropic 在 2024 年看到 Function Calling 痛点后提出 MCP。

### 2.5 Era 5: 单智能体协议（2024-2025）—— 1.5 年

| 年份 | 事件 | 影响 |
|------|------|------|
| **2024-11** | **MCP 发布**（Anthropic）| **工具调用事实标准** |
| 2025-01 | LangChain 集成 MCP | 生态扩张 |
| 2025-04 | **A2A 发布**（Google）| **智能体间协议** |
| 2025-06 | Linux Foundation 接管 A2A | 中立化 |
| 2025-09 | MCP 9700 万下载 | 事实标准 |
| 2025-12 | A2A 50+ 组织 | 联盟形成 |

**关键洞察**：MCP 用 11 个月成为事实标准。**A2A 用 8 个月获得 50+ 组织支持**——速度惊人。

### 2.6 Era 6: 智能体协作（2025-2026）—— 1 年

| 年份 | 事件 | 影响 |
|------|------|------|
| 2025-05 | **IBM ACP** 加入 LF | 商业协议 |
| 2025-06 | Google **UCP** | 通用商务 |
| 2025-10 | **NANDA**（MIT Raskar）| 跨平台发现 |
| 2026-04 | **AIN 草案**（IETF）| 标准化 |
| 2026-04 | **DNS-AID** 草案 | 命名层 |
| 2026-05 | **中国《实施意见》** | 政府推动 |
| 2026-06 | **GB/Z 185** 发布 | 国家强制 |
| 2026-06 | **Verisign 实证**（119,757 端点）| 物理层验证 |

**关键洞察**：智能体协议正在**协议层 + 网络层**双向突破。预计 2027 完成基础协议栈稳定。

### 2.7 Era 7: 智能体互联网（2026+）—— 未来 3-5 年

| 年份 | 预期事件 | 状态 |
|------|---------|------|
| 2026 | MCP/A2A 1.0 LTS | ✅ 已有 |
| 2027 | AIN → RFC | 🔄 进行中 |
| 2027-2028 | DNS-AID / ANS 标准化 | 🔄 进行中 |
| 2028 | `.agent` 顶级域（ICANN）| ⏳ 倡导中 |
| 2028 | Agent Internet 区域互通 | ⏳ 规划中 |
| 2029+ | Agent Internet 全球互联 | ⏳ 长期 |

**关键洞察**：2026-2028 是 Agent Internet 标准化关键窗口期，**错过将形成 5-10 年技术锁定**（类比 1983 年前 hosts 文件时代）。

## 3 三大学习

### 3.1 学习 1：标准化窗口期

| 协议 | 标准化窗口 | 错过的后果 |
|------|----------|-----------|
| DNS（1983） | 5 年 | 不可想象 |
| HTTP（1996） | 10 年 | 浏览器大战 |
| MCP（2024） | **2-3 年** | ？ |
| A2A（2025） | **3-5 年** | ？ |
| AIN（2026） | **2-3 年** | ？ |

**结论**：智能体协议标准化窗口期已打开，**2025-2028 是黄金 3 年**。

### 3.2 学习 2：技术复制的精准性

| 互联网层 | 智能体对应 | 时间差 |
|---------|----------|--------|
| DNS（命名） | ANS / DNS-AID | 43 年 |
| BGP（路由） | AIN | 36 年 |
| TLS（认证） | DANE / TLSA | 31 年 |
| HTTP（消息） | MCP / A2A | 30 年 |
| 商业关系 | ACP / UCP | 30+ 年 |

**结论**：智能体互联网是**互联网基础设施的精准复制**，复用 40 年成熟技术。

### 3.3 学习 3：网络效应曲线

互联网用户从 0 到 10 亿用了 30 年。**智能体数量从 0 到 100 万只用了 2 年**（2024-2026）。

启示：
- 智能体增长是**指数级**
- 标准化时间窗口被**压缩**
- 错过窗口期后果更严重

## 4 与 1983 DNS 标准的对比

| 维度 | 1983 DNS | 2026 Agent Internet |
|------|---------|-------------------|
| 解决的问题 | hosts 文件 N² | 智能体 N² |
| 标准化时间 | 5 年 | **2-3 年** |
| 主导方 | IETF + 政府 | LF + LF + IETF + 国家 |
| 物理层 | UDP/IP | UDP/IP（复用）|
| 命名空间 | 域名 | Agent Domain |
| 路由 | 静态 | 动态 |
| 安全 | DNSSEC | DANE + DNSSEC |

**结论**：Agent Internet 解决**完全相同的问题**（N² 互联），但**技术更复杂**（动态路由 + 智能能力），**时间窗口更短**（2-3 年 vs 5 年）。

## 6 对 v4.0 论文的贡献

本章在 v4.0 论文中起 3 大作用：

1. **历史定位**：说明 Agent Internet 不是"突然出现"，而是 60 年技术演化的必然
