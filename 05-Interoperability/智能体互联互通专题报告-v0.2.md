## 一、问题陈述

### 1.1 智能体"封闭世界"问题

2025-2026 年间，几乎所有主流 AI 平台都发布了智能体框架（Anthropic、OpenAI、Google、Microsoft、Meta），但这些智能体**默认只能在自家平台内协作**。一个 Claude 智能体无法直接发现并调用一个 Gemini 智能体。

**N² 双边集成问题**：5 个平台 = 25 条对接；100 个平台 = 10,000 条对接——这正是 1983 年互联网"hosts 文件时代"的复刻。

### 1.2 历史镜鉴

| 互联网时代 | 关键基础设施 | 标准化窗口 |
|----------|------------|----------|
| 1983 | DNS | 1983-1987 |
| 1989 | BGP | 1989-1994 |
| 1995 | SSL/TLS | 1995-1999 |
| 1996 | XML/SOAP | 1996-2003 |
| **2026** | **智能体协议簇** | **2025-2028（窗口期）** |

## 三、典型方案深度对比

### 3.1 协议层：MCP vs A2A vs AIN

| 维度 | MCP | A2A | AIN |
|------|-----|-----|-----|
| 目标 | 智能体 ↔ 工具 | 智能体 ↔ 智能体 | 智能体 ↔ 智能体（路由）|
| 类比 | ODBC / JDBC | SMTP | BGP / IP |
| 状态 | 有状态（连接会话）| 短状态（任务级）| 无状态（数据报）|
| 拓扑 | 客户端-服务器 | 多对多 | 多对多（路由发现）|
| 成熟度 | 9700 万下载 | 150+ 组织 | 草案阶段 |
| 治理 | Anthropic + 社区 | Linux Foundation | IETF NMRG |

### 3.2 网络层：ANS vs DNS-AID vs AGNTCY

| 维度 | ANS v2 | DNS-AID | AGNTCY |
|------|--------|---------|--------|
| 架构 | DNS + 外部 registry | 纯 DNS | 中心化目录 |
| Lookup 次数 | 多次 | 1 SVCB + 1 DANE | 1 HTTPS |
| 延迟 | 100ms-1s | 1-10ms | 50-200ms |
| 去中心化 | 中 | 高 | 低 |

**Verisign 2026-06 实证**：P90 智能体元数据 = **940 字节**，单条 DNS UDP 响应可承载（1232 字节上限）。

## 四、Verisign 实证：智能体元数据天然适合 DNS

> **来源**：Seethiraju et al. (Verisign) · arXiv:2606.02314v1 · 2026-06-01

### 三维评估框架
1. **Navigational Completeness** — 元数据完整性
2. **Lookup Complexity** — 解析次数
3. **Transaction Performance** — UDP ms 级 vs TCP 多 RTT

### 119,757 端点测量结果
- 端点 URL 中位 70 字节，P90 108 字节
- MCP 工具数中位 3/服务器，P90 19/服务器
- **完整 P90 元数据：609 字节**（不含 DNSSEC）
- + DNSSEC（RSA-2048）：**905 字节**
- + DANE SHA-256：**940 字节**
- **DNS UDP 上限 1232 字节** → 单条消息可承载 ✅

### 三大启示
1. **智能体互联互通的物理层已被互联网解决**
2. **元数据规模天然适配 DNS UDP 消息**
3. **不应发明新传输协议，应复用 40 年互联网基础设施**

## 五、互联网基础设施的"四十年复制"

| 互联网（人类）| 智能体（AI）| 解决的核心问题 |
|------------|------------|--------------|
| DNS（1983-）| ANS / DNS-AID / .Agent | 命名 + 寻址 |
| BGP（1989-）| AIN / AgentDNS | 跨域路由 |
| TLS（1995-）| DANE / 智能体身份验证 | 加密 + 认证 |
| HTTP（1996-）| MCP / A2A | 应用协议 |
| 信用卡（1950-）| ACP / UCP | 商业交易 |

**核心洞察**：智能体互联网不是"发明新互联网"，而是"在 40 年互联网基础设施之上插入智能体层"。

### 6.1 产业建议

- 采用 MCP 作为工具层标准

- 建立智能体能力市场

#### 长期（3-5 年）
- 行业垂直智能体协议
- 跨链 / 跨云智能体迁移
- 智能体间经济模型成熟

### 6.2 政策建议

#### 最紧迫
- 🔥 **推动 `.agent` 顶级域进入 ICANN 议程**

- 智能体责任归属法律框架

## 九、Agent Internet 概念框架（v0.2 新增）

> **核心创新**："Agent Internet" 作为一个独立概念提出，类比"Internet of Things" (IoT) 的命名传统。

### 9.1 定义

**Agent Internet** = 由异构 AI 智能体组成的、可全球互联互通的、基于现有互联网基础设施的"智能体网络"。

### 9.2 与其他概念的区别

| 概念 | 范围 | 关键差异 |
|------|------|---------|
| **Multi-Agent System (MAS)** | 单组织/单平台 | 封闭世界 |
| **Agentic AI** | AI 决策范式 | 强调能力，不强调互联 |
| **AI Agent Ecosystem** | 商业生态 | 商业视角 |
| **Agent Internet** | 全球基础设施 | **强调全球互联互通** |

### 9.3 三层 + 三大基础设施

```
        应用层 (Application)
              │
    ┌─────────┼──────────┐
    │         │          │
  业务层    协议层     网络层
  ACP/MCP/A2A  ANS/DNS-AID
    │         │          │
    └─────────┼──────────┘
              │
    ┌─────────┴──────────┐
    │  智能体互联网骨干网   │
    │  (Agent Internet Backbone)
    │  = IPv6 + 智能体路由  │
    └────────────────────┘
              │
        物理互联网（TCP/IP, DNS, BGP, TLS）
```

### 9.4 Agent Internet 的 4 大特征

1. **全球可达** — 任何智能体可被任何其他智能体发现
2. **可信身份** — 智能体身份可被加密验证
3. **能力描述** — 智能体能机器可读地描述自己
4. **经济结算** — 智能体间可自动支付

## 十、决策树速查（v0.2 新增）

> 详见 [decision-tree-mcp-vs-a2a.md](./decision-tree-mcp-vs-a2a.md)（6.9K）

```
问 1：需要调用外部工具吗？
├── 否 → 无需协议
└── 是 → 用 MCP

问 2：需要其他智能体协作吗？
├── 否 → 仅 MCP
└── 是 → 用 A2A

问 3：协作在同一平台吗？
├── 是 → 平台原生 API
└── 否（跨平台）→ A2A 必选

4 句话核心：
  MCP = 智能体用什么工具
  A2A = 智能体和谁协作
  ANS/DNS-AID = 智能体在哪
  ACP = 智能体怎么交易
```

### 7 项 GB/Z 185-2026 国家标准

| 编号 | 名称 | 类比国际 | 中国独创性 |
|------|------|---------|-----------|
| 185.1 | 总体架构 | A2A 概览 | 顶层设计 |

### AIP 协议（Agent Interconnection Protocol）

**首次写入国家级文件**：2026-05-08《智能体规范应用与创新发展实施意见》

| 维度 | 中国 AIP | 国际 A2A/MCP |
|------|---------|------------|
| 治理 | 国家统筹 | 基金会 / 企业联盟 |
| 身份 | 强制实名 + 国家编码 | 自选 |
| 合规 | 强（备案+审计）| 弱（自律）|
| 跨境 | 需对接 | 天然开放 |
