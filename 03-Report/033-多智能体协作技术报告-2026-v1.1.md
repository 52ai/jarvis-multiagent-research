## 🆕 §九、涌现行为的量化：从观察走向度量

> *本章基于 Riedl (2026, ICLR Poster) "Emergent Coordination in Multi-Agent Language Models"*

### 9.1 核心问题

**多智能体 LLM 系统到底是"独立个体集合"还是"有高层结构的整合集体"？**

传统方法只看任务表现（accuracy / quality score），**无法区分**：
- "涌现协同"（emergent synergy）
- "虚假耦合"（spurious temporal coupling）
- "独立表现聚合"（performance aggregation）

### 9.2 TDMI 信息论框架

Riedl 提出 **Time-Delayed Mutual Information（TDMI）** + Partial Information Decomposition：

| 度量 | 含义 | 量化什么 |
|------|------|---------|
| **TDMI** | 跨 agent 时间延迟互信息 | agent 间的时间耦合强度 |
| **Redundancy** | 冗余信息 | 多 agent 重复表达的内容 |
| **Synergy** | 协同信息 | 只有"合起来看"才存在的信息 |
| **Uniqueness** | 独特信息 | 单 agent 独有的贡献 |

**涌现判据**：**Synergy > 0 且 Identity-linked differentiation 显著** → 真涌现

### 9.3 三组干预实验（Riedl, 2026）

| 实验 | 干预 | 涌现结构 | TDMI 模式 |
|------|------|---------|----------|
| **G1 Control** | 无 prompt 引导 | ❌ 弱协同 | 强时间耦合，**无目标对齐** |
| **G2 + Persona** | "你是经济学家/工程师/哲学家" | 🟡 中等 | 身份差异化，**无互补** |
| **G3 + Persona + ToM** | "考虑其他 agent 可能怎么回答" | ✅ **强涌现** | 身份差异化 + 目标导向互补 |

### 9.4 对 033 课题的启示

> "Effective performance requires both alignment on shared objectives and complementary contributions across members." — Riedl, 2026

1. **涌现可被 prompt 设计触发** — 单纯"加 agent"无效，需"persona + 心理理论提示"
2. **量化涌现成为可能** — TDMI 提供客观度量，告别"看现象说涌现"
3. **与 OWL 框架的关系** — OWL 解决"架构如何让 agent 协同"，Riedl 解决"协同是否真的发生"
4. **Phase 3 涌现专题** — 可直接复现 Riedl 的"猜测游戏 + 3 组干预"实验

### 9.5 Phase 3 涌现实验设计（预告）


## 🆕 §十、Google DeepMind 180 配置实验：可复现的工程基准

> *本章基于 Google DeepMind (2026-01) "Towards a Science of Scaling Agent Systems" (arXiv:2512.08296)*

### 10.1 核心结论

**"多智能体 ≠ 更好"是错的——多智能体的效果高度依赖任务结构。**

| 任务类型 | 多智能体效果 | 关键数据 |
|---------|------------|---------|
| **顺序任务**（planning）| ❌ **下降 39-70%** | 通信开销 > 任务增益 |
| **可并行任务**（金融分析）| ✅ **提升 81%** | 通信开销 << 任务增益 |
| **拐点** | 7+ agent | 协调成本超过收益 |
| **甜区** | **3-5 agent** | 质量/成本最优 |

### 10.2 实验方法

- **规模**：180 种 agent 配置
- **架构变体**：单 agent / 主从 / 平等 / 流水线
- **任务集**：5 类典型多智能体任务（planning / analysis / coding / dialogue / retrieval）
- **预测模型**：87% 准确率判断"何时用多智能体"

### 10.3 与 v1.0 报告的对照

v1.0 报告核心发现 #1 数据来源即本文：

| v1.0 表述 | Google DeepMind 原文 | 一致性 |
|----------|---------------------|--------|
| 顺序任务 -39~70% | Sequential tasks show 39-70% degradation | ✅ 完全一致 |
| 并行任务 +81% | Parallel tasks show 81% improvement | ✅ 完全一致 |
| 3-5 agent 甜区 | 3-5 agents optimal | ✅ 完全一致 |
| 7+ agent 拐点 | >7 agents coordination cost > benefit | ✅ 完全一致 |

### 10.4 对 033 Phase 2 实验设计的指导

`02-Experiments/benchmarks/design-v0.1.md` 12 组 × 5 任务 × 3 次实验矩阵的设计，**直接对齐** Google DeepMind 的方法学：

| 033 设计 | Google DeepMind 对应 |
|---------|---------------------|
| 1 基线 + 5 平等 + 4 主从 + 2 流水线 = 12 组 | 180 配置中 12 类架构 |
| 5 任务 × 3 次 = 15 次/组 | 每组多次重复取中位数 |
| 4 维质量分（事实/结构/可读/洞察）| LLM-as-judge 多维评分 |
| Token 成本 / 通信开销 | 通信开销百分比 |

**结论**：033 实验设计与 Google DeepMind 同源，可作为该论文的**中文任务复现**。
